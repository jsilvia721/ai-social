# sst.config.ts excluded from TypeScript ‚Äî IaC errors not caught at compile time

## Problem Statement

`sst.config.ts` was added to `tsconfig.json`'s `exclude` array to silence type errors from missing SST platform type definitions. This means TypeScript never checks the IaC configuration file. Mistakes in `sst.config.ts` (wrong property names, type mismatches, undefined references) will only surface at `sst deploy` time ‚Äî slow feedback compared to a compile-time error.

## Findings

- **File:** `tsconfig.json` ‚Äî `"sst.config.ts"` in `exclude` array
- **File:** `sst.config.ts` ‚Äî uses `.sst/platform/config.d.ts` triple-slash reference which should provide types
- Root cause: The `.sst/` directory is gitignored, so the type definitions don't exist until `npx sst install` or `npx sst dev` is run
- Confirmed by: Architecture Strategist

## Proposed Solutions

### Option A: Run `npx sst install` in CI before type-check (Recommended)
- In the `lint` job, add a step to run `npx sst install` before `npm run typecheck`
- This generates `.sst/platform/config.d.ts` and enables full type checking of `sst.config.ts`
- Remove `sst.config.ts` from `tsconfig.json` exclude
- Pros: Full type safety on IaC; errors caught in PR CI
- Cons: Adds ~10-20s to lint job; requires AWS credentials or SST's offline mode
- Effort: Small | Risk: Low

### Option B: Create a stub type file for SST globals
- Add `sst.d.ts` to the project with minimal type stubs for `$config`, `sst`, `$interpolate`, `$app`
- Pros: No CI changes needed; works without AWS credentials
- Cons: Manual type stubs diverge from real SST types over time; only partial coverage
- Effort: Small | Risk: Medium

### Option C: Accept exclusion, document why
- Keep `sst.config.ts` excluded; add a comment explaining the reason
- Pros: No change
- Cons: IaC config remains untyped
- Effort: None | Risk: Low (SST deploy fails fast with clear errors)

## Recommended Action

Option B as quick win (type stubs for the SST globals used). Option A if a full CI SST install step is acceptable.

## Technical Details

- **Affected files:** `tsconfig.json`, `sst.config.ts`
- SST type reference: `.sst/platform/config.d.ts` (generated by `npx sst install`)
